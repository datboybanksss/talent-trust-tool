## Task A2: Google OAuth Sign-In

### What
Add a "Continue with Google" button to the `/auth` page on both the Sign In and Sign Up tabs. Uses Lovable Cloud's managed Google OAuth — no external setup needed.

### How

1. **Configure Social Auth** — Use the Lovable Cloud social login tool to generate the `src/integrations/lovable/` module and install `@lovable.dev/cloud-auth-js`.

2. **Create `GoogleSignInButton` component** (`src/components/auth/GoogleSignInButton.tsx`)
   - Imports `lovable` from `@/integrations/lovable/index`
   - Calls `lovable.auth.signInWithOAuth("google", { redirect_uri: window.location.origin })`
   - Handles `result.error` (toast), `result.redirected` (return), and success (session set automatically)
   - Renders a Button with Google icon and "Continue with Google" text
   - Uses existing button styling (no custom variant)

3. **Update `Auth.tsx`**
   - Add `GoogleSignInButton` above the email/password form in both Sign In and Sign Up tabs
   - Add an "or" divider between the Google button and the email form
   - After Google OAuth success, the user enters the same role-based redirect flow (handled by existing `useAuth` + `useEffect` watching `user`)

4. **No database changes** — Google OAuth users get a profile via the existing `handle_new_user` trigger, and a default role via `create_default_subscription`.
