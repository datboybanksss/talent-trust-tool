

## Task A2: Add Google OAuth Sign-In

### What
Add a "Continue with Google" button to the `/auth` page (both Sign In and Sign Up tabs) using Lovable Cloud's managed Google OAuth — no external Google Cloud Console setup needed.

### How

1. **Configure Social Auth** — Use the Lovable Cloud social login tool to generate the `src/integrations/lovable/` module and install `@lovable.dev/cloud-auth-js`.

2. **Create `GoogleSignInButton` component** (`src/components/auth/GoogleSignInButton.tsx`)
   - Calls `lovable.auth.signInWithOAuth("google", { redirect_uri: window.location.origin })`
   - Handles error (toast), redirect, and success states
   - Renders a Button with Google icon and "Continue with Google" text

3. **Update `Auth.tsx`**
   - Add `GoogleSignInButton` above the email form in both tabs
   - Add an "or" divider between the Google button and email fields
   - After Google sign-in, the existing `useEffect` watching `user` handles redirect

4. **No database changes needed** — Google users get a profile via the existing `handle_new_user` trigger automatically.

### Files
- `src/components/auth/GoogleSignInButton.tsx` — new
- `src/pages/Auth.tsx` — modified (add button + divider)
- `src/integrations/lovable/` — auto-generated by tool
